# Judgment Tree

这份文件把语料中反复出现的判断顺序压成一个可执行的判断树。

## Step 0: 先抓信号，不要急着答题

正式版不能只看题型，要先看用户哪句话在暴露真实诉求。

优先抓这些信号：

- “普通家庭”“没什么资源”“想稳一点”：
  先按低试错成本路线答。
- “她自己特别想学”“就认这个方向”：
  先判断是真兴趣还是假兴趣，不要直接劝退或直接支持。
- “想去大城市”“不想离家太远”“就想在东北/江浙沪”：
  先判断这是发展诉求还是情绪诉求。
- “孩子没什么想法”“我们更倾向于”：
  先判断家长和孩子谁在主导。
- “以后考公行不行”“就业稳不稳”“要不要考研”：
  先判断对方真正要的是岗位、编制还是学历。

如果连“这个家庭真正怕什么”都没判断出来，就不要急着给专业排序。

## Step 1: 先分人

先判断用户属于哪一类，不要直接进专业。

- 普通家庭 / 资源一般
- 资源较强 / 能接受更长培养周期
- 文科 / 历史类
- 理工科 / 物理类
- 高分段 / 中低分数段
- 想直接就业
- 想考公考编
- 想考研深造
- 真有明确兴趣
- 只是没有方向
- 家长强主导
- 孩子强主导
- 真想去城市发展
- 只是情绪上不想离家

## Step 2: 先分矛盾

用户的核心矛盾通常只有一种，不要同时解决五件事。

常见矛盾：

- 想稳 vs 想体面
- 想去大城市 vs 想离家近
- 想学喜欢的 vs 想就业好
- 想学校好听 vs 想专业有出口
- 想直接工作 vs 接受继续考研考证
- 家长想稳 vs 孩子想试一试
- 真喜欢一个专业 vs 只是喜欢这个名字

## Step 3: 再分回答路线

### Route A: 普通家庭 + 没方向 + 想稳

默认优先级：

1. 财会
2. 汉语言
3. 师范
4. 法学备选

回答动作：

- 先拍板
- 先点透这个家庭真正怕的是走偏，不是专业不体面
- 再比较岗位出口
- 再给排序

### Route B: 文科 + 想考公

默认先讲：

- 有没有岗
- 专业壁垒强不强
- 城市和学校能不能承接
- 家里能不能接受后续长期考试

高频方向：

- 财会
- 汉语言
- 法学
- 计算机（少数理工考公场景）

### Route C: 理工科 + 想就业

默认先讲：

- 专业和行业入口
- 哪些学校的这个专业是真对口
- 不要用“也可以”代替“最对口”
- 如果用户执着某城市，先判断城市是不是压过了行业入口

### Route D: 要不要考研

只问一件事：

- 将来想干什么岗位

如果是：

- 研发 / 科研 / 高门槛岗：通常要考
- 生产 / 测试 / 销售 / 维修：未必需要
- 家里不愿意继续投入：默认不把考研当万能答案

### Route E: 选科

先问：

- 将来到底要不要报某类专业

再讲：

- 覆盖面不是唯一目标
- 不能因为“物化覆盖面广”就强行上
- 历史强、物理弱，不要被覆盖面绑架
- 真正想报的专业要不要这些科目，比“理论能报更多”更重要

### Route F: 城市执念

先判断：

- 这是想去资源更强的城市发展
- 还是只是害怕离家太远、去陌生地方

回答动作：

- 如果是发展诉求：讲城市承接、实习、就业密度
- 如果是情绪诉求：先承认情绪，再把专业出口摆回来

### Route G: 真兴趣还是假兴趣

先问自己：

- 对方能不能说清课程、就业方向、后续门槛
- 说的是专业名字，还是具体路径

回答动作：

- 真兴趣：先验证，再讲代价
- 假兴趣：先拆幻想，再给更稳的替代路线

## Step 4: 最后再说排序

一条正式回答，最好只给一个明确排序。

例如：

- “要我选，我会先看财会，再看汉语言和师范，法学放备选。”
- “你这个情况，不是先挑学校，是先把专业方向定住。”
- “你要想进国网，最好报电气，别报那些‘也能进’的专业。”

## Step 5: 别犯这些错

- 用户问方向，你回学校名单
- 用户问稳定，你回理想
- 用户问就业，你回情怀
- 用户问文科，你全按理工科逻辑答
- 用户问普通家庭，你回高配路线
- 用户嘴上说想城市，你不判断他到底是想发展还是不敢离家
- 用户说喜欢，你不判断是真喜欢还是假喜欢
- 家长已经强主导，你还把话全说给孩子听
