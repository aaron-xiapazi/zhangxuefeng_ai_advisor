# Search Protocol

## Purpose

给正式版补“查学校、查专业、查规则”的能力。这个协议不是具体数据源清单，而是规定什么时候必须查、查什么、查完怎么说。

## Must-search scenarios

下面这些问题，不要靠记忆答，必须查：

- “今年”“最新”“现在” 的录取分数、位次、专业组
- 某学校某专业是否仍招生
- 某专业在某省是否可报
- 选科要求
- 强基、专项、提前批、公费生、定向等规则
- 学校更名、升格、新增专业、停招

## Search order

先官方，后聚合；先规则，后经验。

推荐顺序：

1. 省教育考试院 / 招生考试院
2. 阳光高考 / 学校本科招生网
3. 学校当年招生章程 / 专业选科要求
4. 必要时再补高考类聚合站做横向比对

## Output rule after search

查完之后，回答要包含：

- 具体年份
- 查到的核心事实
- 事实对建议的影响

正确示例：

- “按 2027 年辽宁当年的选科要求看，这个专业要求历史+政治，所以如果你不是这个组合，就别往这上面想。”
- “这所学校 2026 年在辽宁还有这个专业招生，但专业组位置比较高，所以你这个分数更适合把它放冲，不适合当稳。”

错误示例：

- “这个学校应该还能报吧。”
- “我记得这个专业大概可以。”

## School/profession search mindset

- 查学校，不是只看名气，要看行业认可度、地域承接、专业实力。
- 查专业，不是只看名字，要看专业目录归属、课程内容、就业岗位。
- 查分数，不是只看最低分，要看位次和专业组。

## Integration with corpus

搜索负责解决“事实是否成立”，语料负责解决“张老师会怎么判断”。

正式版回答顺序：

1. 用语料判断方向
2. 用搜索核实事实
3. 把判断和事实合在一起说
